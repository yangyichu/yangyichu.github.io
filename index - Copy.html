Latest Projects

Using Reinforcement Learning to play Binding of Isaac

The Binding of Isaac is a randomly generated action RPG shooter with heavy Rogue-like elements. The first thing I did was to write a gym-like environment for interaction. I used win32api and win32gui in python to get screen shot and send control command to inactive window directly(Otherwise I can do nothing on my PC at the same time). Also I used CheatEngine to speed up the game about 20x. To make things easier I modified a mod in LUA called easydebug so that I can quickly reset the game by sending only 1 key command instead of typing a sentence in the console. Next part is the reinforcement agent, two network structure was tested, DQN(deep Q-network) and DRQN (Deep Recurrent Q-Network), they are basically the same except DQRN substitute the first fully-connected layer with a LSTM layer. The frames are collected at 30fps, for DQN we need to wrap the env with a frame-skip and stack class so that the agent can have some sense of time lapse, for DRQN we don't need that. So the grabbed frames are compressed to 82x82 then go through 3 layer of Conv nets and a fully-connected layer(LSTM for DRQN), then to the output layer.

Results of 2M steps on DRQN with 4-actions are shown below. The action space I chose was too large(32, w a s d ↑ ↓ ← → and the combination) at first, the algorithm failed to converge. Then I reduced the action space to only 4 action (w↓,a→,s↑,d←) and the agent successfully learned to beat the boss(Pin, a rather easier boss, initially it was Monstro which is too hard) in 1.5M steps.

Using Reinforcement learning to control robot arm with Hind-sight Experience Replay

The action space of this robotic environment is continuous, so discrete methods suchas DQN will not perform well. So in this project, we used Deep Deterministic Policy Gradient(DDPG) that can handle continuous action space. DDPC follows the Actor-Critic structure, and has a pair of behavioral actor, critic network and a pair of target actor, critic network. As for TD3, it is a modified version of DDPG. TD3 learns two Q-functions instead of one (hence “twin”),and uses the smaller of the two Q-values to form the targets in the Bellman error lossfunctions. TD3 updates the policy (and target networks) less fre-quently than the Q-function. TD3 adds noise to the target action, to make it harder for the policy to exploit Q-function errors by smoothing out Q along changes in action.

Optimization-Free Quadruped robot control using explicit reference governor

Many of today’s real-time approaches have achieved betterefficiencies through reduced-order models and decomposi-tion methods, however, they still depend on optimizationsolvers. While many optimization-based methodsdo not present promising horizons in experimental legged robotics, other Gauss-Newton Hessian approximation suchas Sequential Linear Quadratic (SLQ), iLQR and iLQG have been more effective. That said, they also present formidable challenges for smaller processors. In this work, we will resolve gait parameters for ourquadrupedal platform off-line and re-plan them in real-timeduring the gait cycle. We assume well-tuned supervisory controllers and will focus on fine-tuning the joints desired trajectories to satisfy unilateral contactforce constraints. To do this, we will devise intermediaryfilters based on Explicit Reference Governors (ERG). ERGs relied on provable Lyapunov stability propertiescan perform the motion planning problem in the state spacein a much faster way than widely used optimization-basedmethods.

Development of Quadruped Robot Simulator in Simulink

This is a simulator created in Simulink. I created the URDF file from the Solidworks Assembly of robot, and then configured everything in Simulink SimMechanics with Contact Force Library. Then a FK/IK algorithm is written to test the action space of robot. A simple balancing algorithm is developed based on IK and virtual sensors and it works well in simulation. This model is later used for testing and research such Hardware-in-the-loop simulation. A full system of robot, simulink, Speedgoat and Mocap is then developed based on this framework.

Lidar Mapping and Localization

The data used is collected from a moving vehicle driving around Boston downtown(including data from 2 Lidars, 2 RGB camera, 1 Thermal camera, GPS and 6-axis IMU) and is stored in rosbag format. It is then extracted and converted into point cloud file and csv. Then I implemented ICP registration in MATLAB with sensor data fusing (GPS, IMU and Lidar) based on complementary filter and Kalman filter to make the registration more accurate. I also take advantage of the preprocessing method used in LeGO-LOAM to compensate for Lidar distortion on a moving vehicle. To make a cleaner dense map, ground, buildings and vehicles are then extracted from the original map using Euclidean clustering. This project is at first written in MATLAB, and is later re-implemented in C++, to take advantage of the PCL library, which allows for faster frame processing(The original dataset is hours long and could take a day to process in MATLAB, with C++ it is much faster).
